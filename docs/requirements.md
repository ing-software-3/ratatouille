# RF
## Sistema

RF01. Sistema de visualización de comidas rápidas mediante una página web interactiva.

RF02. Sistema de recomendaciones personalizadas de comidas rápidas basado en preferencias del usuario.

RF03. Sistema de menú para visualizar comidas rápidas más populares y mejor calificadas.

RF04. Sistema de registro e inicio de sesión de usuarios.

RF05. Sistema de recuperación de contraseña mediante correo electrónico.

RF06. Sistema de actualización automática de información de comidas, precios y disponibilidad.

RF07. Sistema de guardado de comidas rápidas favoritas por parte del usuario.

RF08. Sistema de registro de productos alimenticios con información de nombre, ingredientes, categoría y precio.

RF09. Sistema para editar y eliminar productos o comidas rápidas del catálogo.

RF10. Sistema para mostrar disponibilidad de productos y restaurantes en tiempo real.

RF11. Sistema de búsqueda avanzada de comidas rápidas por nombre, categoría, ingredientes o precio.

RF12. Sistema de categorías de alimentos (entradas, postres, platos fuertes, bebidas, combos, etc.).

RF13. Sistema generador de alertas cuando un producto se encuentre agotado o con bajo inventario.

RF14. Sistema para agregar nuevos productos y restaurantes al catálogo.

RF15. Sistema de calificación y comentarios sobre comidas rápidas y restaurantes.

RF16. Sistema de análisis inteligente mediante aprendizaje automático para mejorar recomendaciones.

RF17. Sistema de filtrado de comidas según restricciones alimenticias, alergias o ingredientes no deseados.

RF18. Sistema de historial de búsquedas y recomendaciones recientes del usuario.

RF19. Sistema administrativo para gestionar usuarios, productos y recomendaciones.

RF20. Sistema de estadísticas sobre comidas más buscadas, populares y mejor calificadas.

RF21. Sistema de notificaciones para informar promociones, nuevos productos o recomendaciones.

RF22. Sistema de comparación de comidas rápidas según precio, ingredientes o calificaciones.

RF23. Sistema de carga de imágenes para productos y restaurantes.

RF24. Sistema de integración con APIs externas de restaurantes o plataformas de domicilios.

RF25. Sistema de detección de preferencias alimenticias basado en comportamiento del usuario.

RF26. Sistema de recomendaciones en tiempo real basado en tendencias y consumo de otros usuarios.

RF27. Sistema de validación de datos para evitar registros duplicados o información incorrecta.

RF28. Sistema de roles y permisos (administrador y usuario).

RF29. Sistema de generación de reportes sobre inventario, usuarios y recomendaciones.

RF30. Sistema de cierre de sesión seguro y control de sesiones activas.

## NRF

NRF01. El sistema deberá soportar múltiples peticiones simultáneas sin superar el 80% del uso de recursos del servidor.

NRF02. El sistema deberá generar sugerencias automáticas de comidas del día.

NRF03. El sistema deberá facilitar la toma de decisiones mediante una interfaz intuitiva y amigable.

NRF04. El tiempo de respuesta de carga de páginas no deberá superar 1 minuto.

NRF05. El sistema de login deberá garantizar autenticación segura mediante cifrado de contraseñas.

NRF06. El sistema deberá ser fácil de usar para usuarios y personal administrativo.

NRF07. Las consultas y recomendaciones deberán responder en menos de 2 segundos.

NRF08. Solo usuarios autorizados podrán modificar inventario, productos o configuraciones administrativas.

NRF09. El sistema deberá permitir agregar nuevos productos y usuarios sin afectar el rendimiento general.

NRF10. El sistema deberá funcionar en los principales navegadores web mediante integración con API desarrollada en FastAPI.

NRF11. El código fuente deberá estar documentado y organizado para facilitar mantenimiento y futuras mejoras.

NRF12. El sistema deberá garantizar la disponibilidad y consistencia de la información almacenada.

NRF13. El sistema deberá ser escalable para soportar crecimiento en usuarios y datos.

NRF14. El sistema deberá mantener copias de seguridad automáticas y periódicas de la base de datos.

NRF15. El sistema deberá proteger la privacidad de los datos personales de los usuarios.

NRF16. El sistema deberá contar con una interfaz adaptable a dispositivos móviles, tablets y computadores (responsive).

NRF17. El sistema deberá garantizar una disponibilidad mínima del 95% del tiempo.

NRF18. El sistema deberá permitir mantenimiento y actualizaciones sin afectar significativamente el servicio.

NRF19. El sistema deberá registrar errores y actividades del sistema mediante logs.

NRF20. El sistema deberá implementar validaciones para evitar inyecciones SQL y accesos no autorizados.

NRF21. El sistema deberá tener compatibilidad con bases de datos relacionales y no relacionales.

NRF22. El sistema deberá optimizar el consumo de recursos del servidor y la base de datos.

NRF23. El sistema deberá permitir futuras integraciones con inteligencia artificial y nuevos modelos de recomendación.

NRF24. El sistema deberá garantizar accesibilidad básica para usuarios con dificultades visuales.

NRF25. El sistema deberá mantener tiempos de respuesta estables aun con incremento de usuarios concurrentes.
